# 修士論文発表支援プロンプト：発表準備・質疑応答テンプレート

**【重要：このプロンプトの適切な使用について】**

このプロンプト集は、公聴会での質疑応答に備えて**思考を深めるため**のツールです。以下の点を必ず理解した上で使用してください。

- **想定質問はサンプルに過ぎません**：AIが生成する質問は、あくまで「こういう観点の質問が来る可能性がある」という例示です。実際の公聴会では、これと全く異なる質問が出されることが普通です。特定の質問への「正解」を準備することではなく、研究全体への理解を深めることが目的です。

- **回答は必ず自分で考えてください**：テンプレート3で「自作回答の評価」を依頼する際も、AIには回答の改善方向を示させるだけで、完成した回答文を書かせることはしません。公聴会は、あなた自身の研究理解と即応力を評価する場であり、準備された回答を暗記して述べることは学術的誠実性に反します。

- **このツールの正しい使い方**：
  - ✅ 自分が見落としていた質問の観点に気づく
  - ✅ 自作した回答の論理的弱点を確認する
  - ✅ 説明の優先順位や時間配分を検討する
  - ❌ AIが生成した質問リストを「出題予想」として暗記する
  - ❌ AIが示した改善案を丸写しして回答を作る

公聴会の真の目的は、あなたが自分の研究を深く理解し、それを自分の言葉で説明し、予期しない質問にも柔軟に対応できる力を評価することです。このプロンプトは、その準備プロセスを支援するものであり、本番での「カンニングペーパー」を作るためのものではありません。

---

以下のテンプレートから、目的とタイミングに合ったものを選択して使用してください。  
ファイル（スライドPDFなど）を添付する場合、**プレースホルダに何も書かれていない情報は、添付ファイルから読み取る**ことを AI に指示する前提としています。

- テンプレート1：公聴会前の発表構成・内容改善用
- テンプレート2：公聴会前の想定質問シミュレーション用
- テンプレート3：想定質問に対する自作回答の妥当性評価用

想定している公聴会は、数学・データサイエンス分野における **20分程度の発表＋10分程度の質疑応答** です（必要に応じて変更してください）。

## テンプレート1：発表構成・内容の事前チェック（準備段階用）

公聴会用スライドが概ね完成した段階で、発表の構成や説明の流れを改善するために使用します。  
特に、公聴会でよく問われる「研究の位置付け」「仮定の妥当性」「結論の一般性・限界」などの観点から、発表内容を整理し直したい場合に利用してください。

**【前提】**

- 数学またはデータサイエンス分野の修士論文公聴会を想定しています。
- 発表時間は「20分＋10分質疑」を標準としますが、必要に応じて変更してください。
- スライドは、文字情報を含む PDF（例：Beamer から出力された PDF）で用意されていることを想定しています。
- プレースホルダに記入されていない情報は、**添付したスライドPDFから読み取って補完する**ように AI に指示してください。

```markdown
**【入力欄】**

- **発表タイトル**：  
  `[発表タイトルを記載（空欄の場合は、添付スライドから読み取ること）]`

- **発表時間と質疑時間**：  
  例）`発表20分＋質疑10分`  
  `[実際に想定している時間配分を記載（空欄の場合は、20分＋10分として扱うこと）]`

- **対象分野・キーワード**：  
  例）`代数トポロジー / ホモロジー / 永続ホモロジー`  
  `[分野名と主要なキーワードを3〜5個記載（空欄の場合は、添付スライドの内容から推定すること）]`

- **想定聴衆**：  
  例）`同分野の教員2〜3名、隣接分野の教員1〜2名、同研究科の院生`  
  `[想定している聴衆の専門性・人数構成を記載（空欄の場合は、通常の修士公聴会を想定してよいこと）]`

- **スライド構成の概要**：  
  例）`1. 研究背景と目的、2. 理論的準備、3. 提案手法、4. 実験結果、5. まとめと今後の課題`  
  `[セクション構成や各パートの役割を記載（空欄の場合は、添付スライドから構成を読み取ること）]`

- **スライドPDFから抽出したテキストの概要（任意）**：  
  `[全文貼付が難しい場合は要約を記載。空欄の場合は、添付スライドの本文を直接参照すること]`

- **特に不安を感じている点（任意）**：  
  例）`背景の説明が長すぎるかもしれない`, `仮定の説明が十分か不安`  
  `[事前に気になっている点を記載（空欄の場合は、一般的な観点からコメントすること）]`

**【AIへの依頼内容】**

上記の情報と、添付したスライドPDFの内容を踏まえて、以下の観点から発表構成と内容をチェックし、改善のための具体的なコメントを出してください。

1. 公聴会（20分＋10分質疑）として、時間配分と内容のバランスは適切か。  
2. 研究背景・目的・貢献が、専門外の教員にも理解できる程度に明確か。  
3. 数学的・理論的な前提や仮定の説明に抜けや曖昧さがないか。  
4. 提案手法や結果の説明で、「ここは質問されやすい」と考えられる箇所はどこか。  
5. 発表をより分かりやすくするために、「削ってよい部分」「もう少し詳しく説明すべき部分」があれば具体的に指摘すること。
```

## テンプレート2：想定質問シミュレーション（準備・直前確認両用）

公聴会で想定される質問を列挙し、質問者の理解レベル（初心者／中級者／上級者）ごとに整理するためのテンプレートです。  
スライドがある程度できている段階でも、公聴会直前の最終確認としても利用できます。

**【前提】**

- 数学・データサイエンス分野の修士論文公聴会を想定します。
- スライドPDFは、文字情報を含むファイルであると仮定します。
- プレースホルダに記入されていない情報は、**添付したスライドPDFから読み取る**ことを AI に依頼してください。
- 実際の公聴会で頻度が高い「現実的な質問」を優先し、査読報告レベルの極端に細かい指摘は控えてください。

```markdown
**【入力欄】**

- **発表タイトル**：  
  `[発表タイトルを記載（空欄の場合は、添付スライドから読み取ること）]`

- **発表時間と質疑時間**：  
  例）`発表20分＋質疑10分`  
  `[実際に想定している時間配分を記載（空欄の場合は、20分＋10分として扱うこと）]`

- **対象分野・キーワード**：  
  `[分野名と主要なキーワードを3〜5個記載（空欄の場合は、添付スライドの内容から推定すること）]`

- **想定聴衆の構成**：  
  例）`同分野の専門家2名、隣接分野の教員1名、修士学生数名`  
  `[専門家／他分野教員／学生などの構成を記載（空欄の場合は、一般的な修士公聴会の構成を想定してよいこと）]`

- **スライドPDFから抽出したテキストまたは概要**：  
  `[テキストや要約を記載（空欄の場合は、添付スライドの本文を直接参照すること）]`

- **特に質問されそうだと感じている箇所（任意）**：  
  `[自分が懸念しているスライド番号や内容を記載（空欄の場合は、AIがスライド全体から候補を挙げること）]`

**【AIへの依頼内容】**

上記の情報と添付スライドPDFをもとに、以下の条件を満たす質問リストを作成してください。

1. 質問者の理解レベルごとに、質問を分類して列挙すること。  
   - 初心者レベル（基礎概念や全体像の理解を確認する質問）  
   - 中級者レベル（手法の妥当性や実験設定を問う質問）  
   - 上級者レベル（理論的な前提・限界・一般化可能性を深掘りする質問）

2. 各レベルについて、3〜5件程度の質問を提示すること。

3. 各質問ごとに、以下の情報を付けてください。  
   - 質問文  
   - 「この質問で何を確認しようとしているか」という狙いの説明（1〜2文）  
   - 想定される質問者（例：同分野の教員／隣接分野の教員／修士学生 など）

4. 実際の公聴会で現実的に想定される質問を優先し、極端に細かい技術的詳細や、査読者コメントのような要求は控えること。
```

## テンプレート3：想定質問に対する自作回答の妥当性評価（直前確認用）

テンプレート2などで得られた想定質問、あるいは自分で想定した質問に対して、自分で作成した回答案の妥当性を評価するためのテンプレートです。  
AIに「模範解答を書かせる」のではなく、自作回答のどこを改善すべきかを検討するために使用してください。

**【前提】**

- 評価対象となる回答文は、発表者自身が作成したものとします。
- プレースホルダに記入されていない前提情報は、**添付したスライドPDFから読み取る**ように AI に依頼してください。
- AIには、回答文の全面的な書き換えではなく、「どの点を修正・削減・具体化すべきか」という観点でフィードバックするよう依頼します。
- 公聴会での口頭回答であることを考慮し、時間制約（例：30秒〜1分程度）も含めて評価します。

```markdown
**【入力欄】**

- **質問文**：  
  `[想定している質問文を記載]`

- **想定質問者のレベル・立場**：  
  例）`中級者レベルの隣接分野の教員`, `上級者レベルの同分野の教員`  
  `[質問者の専門性を記載（空欄の場合は、一般的な教員レベルを想定すること）]`

- **自分の回答案（口頭で答える想定）**：  
  `[実際に自分で考えた回答文を記載]`

- **回答にかけられる時間の目安**：  
  例）`30秒程度`, `1分以内`  
  `[実際に口頭で話せる長さを想定して記載（空欄の場合は、1分以内で話せることを前提とすること）]`

- **関連するスライドや箇所（任意）**：  
  例）`スライド12の定理2.1に関する質問`  
  `[回答が関連するスライド番号などを記載（空欄の場合は、AIがスライド全体から関連箇所を推定すること）]`

- **評価してほしい観点（任意）**：  
  例）`論理の一貫性を優先して見てほしい`, `専門用語の使い方が適切か確認してほしい`  
  `[特に重視したい評価軸があれば記載（空欄の場合は、総合的に評価すること）]`

**【AIへの依頼内容】**

上記の質問文・回答案・前提条件と、必要に応じて参照されるスライドPDFの内容をもとに、以下の観点から回答案を評価し、改善のヒントを示してください。

1. 論理的一貫性：  
   - 結論と理由付けの関係に飛躍や矛盾がないか。  
   - 前提が暗黙になりすぎていないか。

2. 専門性のレベル感：  
   - 想定質問者のレベルに対して、説明が難しすぎないか／逆に表面的すぎないか。  
   - 専門用語の選び方は適切か。

3. 分かりやすさと構成：  
   - 公聴会での口頭回答として、短時間で要点が伝わる構成になっているか。  
   - 不要に冗長な部分や、逆に説明不足な部分がないか。

4. 時間制約への適合性：  
   - 指定した回答時間（例：30秒／1分以内）で話し切れる分量になっているか。

5. 改善の方向性：  
   - 回答文を丸ごと書き換えるのではなく、  
     - 削ってもよい部分  
     - もう少し具体化した方がよい部分  
     - 順番を入れ替えた方が伝わりやすい部分  
     などを指摘し、「どのように修正すべきか」の方針を簡潔に示すこと。
```
