# 日本語 LaTeX 環境の選択

修士論文を執筆するにあたり、まず「どの種類の LaTeX（エンジン）」を使用するかを決める必要があります。現在の日本語環境は過渡期にあり、ネット上の古い情報に惑わされないことが重要です。

## 1. 日本語 LaTeX の現状
日本語を扱うための LaTeX には、大きく分けて以下の2つの流れがあります。

- **(u)pLaTeX**: 
    - 長らく日本国内の標準だった環境です。
    - ネット上に膨大なノウハウやテンプレートが存在しますが、PDF作成に「DVI」という中間ファイルを経由する古い仕組みを必要とします。
- **LuaLaTeX (現在の推奨)**: 
    - 現代的に再設計されたエンジンで、世界的な標準に最も近いです。
    - 直接 PDF を出力でき、OS標準のフォント（ヒラギノ、MS明朝など）の扱いが非常に容易です。

## 2. なぜ LuaLaTeX を推奨するのか
特にこだわりがない限り、本キットでは **LuaLaTeX** の利用を推奨します。

- **最新のパッケージ対応**: 最新の数学系パッケージや図表操作パッケージの多くは LuaLaTeX を前提に開発されています。
- **数学記号の親和性**: Unicode にネイティブ対応しているため、`unicode-math` 等を利用することで、特殊な数学記号の表示トラブルが激減します。
- **クラウド環境との相性**: Overleaf や CloudLaTeX, Prism などのクラウド環境で標準的にサポートされており、環境構築の手間が最小限で済みます。

## 3. エンジン間の互換性
(u)pLaTeX 用に書かれたソースコードも、多くの場合、冒頭の「プレアンブル」を数行書き換えるだけで LuaLaTeX に対応させることができます。

### 設定の比較例

| 設定項目 | (u)pLaTeX | LuaLaTeX (推奨) |
| :--- | :--- | :--- |
| **クラス指定** | `\documentclass[uplatex]{jsarticle}` | `\documentclass{ltjsarticle}` |
| **フォント設定** | `\usepackage[utf8]{inputenc}` (不要な場合も多い) | `\usepackage{luatexja}` (自動で読み込まれることが多い) |

> [!TIP]
> 既に先輩から受け継いだテンプレートが (u)pLaTeX 用である場合は、無理に変更せずそのまま使うのも一つの手です。しかし、新規に作成する場合は LuaLaTeX を選択することで、将来的なトラブルを避けることができます。
