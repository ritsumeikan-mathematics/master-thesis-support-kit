# (参考) Git/GitHub によるバージョン管理

修士論文は数ヶ月から一年にわたる長期プロジェクトです。「昨日書いた内容に戻したい」「PCが壊れた」といったトラブルから論文を守るために、Git によるバージョン管理の導入を推奨します（本章は応用編です）。

## 1. なぜ Git を使うのか
LaTeX はテキストファイルであるため、Word 等のバイナリファイルよりも Git と非常に相性が良いです。

- **執筆履歴の保存**: 「第1章を加筆」「数式を修正」といった単位で記録を残せます。
- **クラウドからの独立**: クラウドエディタ（Overleaf等）と GitHub を連携させることで、万が一サービスが停止しても、GitHub 上に最新の原稿が残ります。
- **差分の確認**: 以前のバージョンとどこが変わったのかを、行単位で明確に比較できます。

## 2. GitHub を活用した応用例

### A. 自動バックアップ
ローカルで執筆している場合でも、`git push` を行うだけで、世界最強のコードホスティングサービスである GitHub に原稿が安全に保管されます。

### B. GitHub Actions による自動PDF生成 (CI)
GitHub に `.tex` ファイルをプッシュすると、クラウド上で自動的に LaTeX が実行され、PDF を生成する仕組み（CI/CD）を構築できます。
- **メリット**: 手元に LaTeX 環境がない端末（スマホやタブレット）からでも、最新の PDF を確認できます。
- **ツール例**: `dante-ev/latex-action` などの GitHub Action を利用するのが一般的です。

### C. Overleaf / Prism との同期
多くのクラウド LaTeX サービスは GitHub との同期機能を備えています。
1. クラウド上で執筆
2. 自動的に GitHub へ同期（バックアップ）
3. 万が一の際はローカルの VS Code で続きを執筆
というハイブリッドな運用が可能になります。

## 3. 推奨される `.gitignore`
LaTeX はコンパイル時に大量の中間ファイル（`.aux`, `.log`, `.toc` 等）を生成します。これらを Git の管理対象から外すため、リポジトリのルートに以下の `.gitignore` を配置することを推奨します。

```text
# LaTeX 中間ファイルの無視設定
*.aux
*.log
*.toc
*.out
*.fdb_latexmk
*.fls
*.synctex.gz
*.pdf
```
