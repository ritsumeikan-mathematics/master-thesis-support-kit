# クラウド実行環境の利用と比較

環境構築の手間を省き、ブラウザだけで執筆を開始できるクラウド環境は非常に強力です。しかし、修士論文という長期プロジェクトにおいては、サービスの持続性やデータ管理に注意を払う必要があります。

## 1. 主要サービスの比較 (2026年1月時点)

| サービス名 | 特徴 | メリット | デメリット |
| :--- | :--- | :--- | :--- |
| [Overleaf](https://www.overleaf.com) | 世界シェア1位 | 共同編集機能が強力。世界中のテンプレートが使える。 | 無料枠では GitHub 連携が不可。日本語設定に工夫が必要。 |
| [CloudLaTeX](https://cloudlatex.io) | 日本企業（アカリク）運営 | 日本語（LuaLaTeX/uplatex）が標準で完璧に動く。国内の学術誌用テンプレートが豊富。 | 海外の研究者との共同編集には不向き。 |
| [Prism](https://prism.openai.com) | OpenAI提供（最新） | **GPT-5.2 統合型**。AIと対話しながら数式や証明を生成可能([解説](https://openai.com/ja-JP/index/introducing-prism/))。 | 比較的新しいサービスのため、長期的な安定性が未知数。 |

## 2. サービス持続性のリスクと対策

クラウドサービスは便利ですが、以下のリスクを常に意識してください。
- **突然のサービス終了**: 運営会社の意向によりサービスが停止する可能性があります。
- **仕様変更**: 無料枠の制限や、対応エンジンの変更によりコンパイルできなくなるリスクがあります。
- **通信障害**: 締切直前にサーバーがダウンし、執筆が止まるリスクがあります。

### 推奨されるリスク管理
1.  **定期的な外部保存**: 月に一度はプロジェクト全体を `zip` 形式でダウンロードし、ローカルやクラウドストレージ（OneDrive, Google Drive等）にバックアップしてください。
2.  **GitHub 連携**: 可能であれば GitHub と連携させ、常に最新の `.tex` ファイルが外部サーバーにも存在する状態を作ってください。
3.  **ローカルとの併用**: [setup-local.md](./setup-local.md) を参考に、手元の PC でもコンパイルできる状態を一度は確認しておくことを強く推奨します。

## 3. クイックスタートガイド

どのサービスを利用する場合も、以下の手順で開始できます。

1. **プロジェクト作成**: 「Blank Project」を選択します。
2. **エンジンの設定**:
    - **CloudLaTeX**: デフォルトで LuaLaTeX が選択可能です。
    - **Overleaf**: `Menu` → `Compiler` から `LuaLaTeX` を選択してください。
    - **Prism**: 設定パネルから `Engine: LuaLaTeX` を確認してください。
3. **日本語の確認**: [setup-local.md](./setup-local.md) にある動作確認用コードを貼り付けてコンパイルし、PDF が生成されるか確認します。
